## 1. デザイン原則（コンセプト → 視覚の翻訳）

1. **キャンバスが主役、UI は黒子** — マインドマップの思考空間が中心。ツール UI は控えめで、思考を妨げない（提供価値「主導権は人間」）
2. **静けさと余白** — 余白広め・低彩度。煽らず急かさず、「整理して離れる」を尊重（charter §2.2）
3. **人と AI を視覚で区別** — 人が書いたノードと AI が提案した枝を色・線で明確に分ける（共同編集の主導権が人間にあることを視覚化）
4. **読みやすさ最優先** — ノードのテキストが主コンテンツ。コントラスト・行間を確保
5. **穏やかな動き** — ライブで「ゆっくり育つ」体験。アニメーションは緩やか、点滅・派手な遷移を避ける
6. **押し付けない** — 通知圧・バッジ・赤い警告を多用しない（中毒性を狙わない）

## 2. カラートークン

| トークン | 値 | 用途 |
|---|---|---|
| `--color-bg` | `#FAF9F6` | アプリ背景（紙のような off-white） |
| `--color-surface` | `#FFFFFF` | ノード面・カード・パネル |
| `--color-surface-muted` | `#F2F0EA` | サブパネル・hover 面 |
| `--color-text` | `#2A2E35` | 本文（インク） |
| `--color-text-muted` | `#6B7280` | 補助テキスト・プレースホルダ |
| `--color-primary` | `#3B6E8F` | 主色（落ち着いたインクブルー、主要操作・人ノードの強調） |
| `--color-primary-hover` | `#33617E` | primary hover |
| `--color-accent` | `#5FA8A0` | アクセント（柔らかティール = AI 枝提案） |
| `--color-accent-soft` | `#E3F0EE` | AI 提案の薄い面 |
| `--color-border` | `#E5E2DB` | 境界・ノード枠 |
| `--color-success` | `#5B9279` | 成功（くすんだ緑） |
| `--color-warning` | `#C9994A` | 注意（琥珀、赤を避ける） |
| `--color-danger` | `#B5654A` | 破壊操作（くすんだテラコッタ、削除等。鮮やかな赤は使わない） |

> **人 vs AI の視覚区別**: 人ノード = `--color-surface` + `--color-border` 実線。AI 提案ノード = `--color-accent-soft` 面 + `--color-accent` **点線**枠（承認すると実線・人ノード化）。

## 3. タイポグラフィ

| 役割 | 指定 | サイズ/行間 |
|---|---|---|
| フォント | 中立サンセリフ（system-ui / "Inter" / "Noto Sans JP" フォールバック） | — |
| ノードテキスト | regular | 14–15px / 1.6 |
| 見出し（パネル） | medium | 16–18px / 1.4 |
| 補助・キャプション | regular | 12px / 1.5 |

- Web フォント: 最小限（system-ui 優先で読込ゼロ可、日本語は Noto Sans JP を必要時のみ）

## 4. 形・影・余白

| トークン | 値 |
|---|---|
| `--radius-sm` | 4px |
| `--radius-md` | 8px（ノード・カード） |
| `--radius-lg` | 12px（パネル） |
| `--shadow-soft` | `0 1px 3px rgba(42,46,53,0.06)`（薄い影） |
| `--shadow-panel` | `0 4px 16px rgba(42,46,53,0.08)` |
| 余白スケール | 4 / 8 / 12 / 16 / 24 / 32 / 48px |

- 角丸控えめ・影薄め。線は細く（1px border）。装飾は line-art。

## 5. コンポーネント（基本形）

- **ボタン**: primary（塗り `--color-primary`）/ secondary（枠線）/ ghost（透明）/ danger（`--color-danger`）。角丸 `--radius-md`、medium テキスト
- **ノード（キャンバス）**: 白面 + 細枠 + `--shadow-soft`。人ノード=実線、AI 提案=ティール点線。選択時 primary 枠
- **チップ**: 枝種別（関連/対立/問い/具体例）を低彩度の色分けチップで
- **入力**: 枠線 + focus で primary リング。プレースホルダは `--color-text-muted`
- **パネル/ドロワー**: `--color-surface` + `--shadow-panel` + `--radius-lg`（AI 提案リスト・設定）
- **ヘッダー（モバイル横一列標準形）**: アプリ名(左) + コンパクトメニュー(右) を `white-space:nowrap` + `flex-shrink:0` で行内折り返し禁止。ブランド名は `min-width:0`+`overflow:hidden`+`text-overflow:ellipsis` で 1 行省略。収まらないメニューはアイコン/オーバーフロー(…)へ退避（狭幅で縦積み/縦書きに見せない、CF-20260609-007）
- **入口の「これは何？」導線（O41）**: 入口画面に簡潔なリード文を優先配置。収まらない場合は常設インフォボタン（丸付き「?」）+ 軽量モーダルで「会議/講義を聞きながら AI と往復で考えをほどくマインドマップ」を説明
- **進捗体験（O45）**: AI 逐次マージ/枝提案の本質遅延（>1–2s）は素のスピナーでなく「実段階に紐づくブランド文言 + 軽い動き」で（例: 「要点を聞き取っています…」→「枝を整えています…」）。嘘進捗・わざと遅延は NG

## 6. ボイス & コピー（O38 準拠）

- ユーザー向け文字列に技術用語を入れない（NG: 「ノードを永続化」「API エラー」「トークン超過」）
- 言い換え例: 「保存しました」「うまくつながりませんでした。もう一度お試しください」「今月の AI のお手伝いを使い切りました（100円で追加できます）」
- トーン: 静か・やさしい・押し付けない。命令形より提案形（「広げてみますか?」）
- エラー: 責めない・原因を平易に・次の一手を示す
- 最終的な文言の声は `/flow:wording`（O42 対話校正）で仕上げる

## 7. アイコン & イラスト & ブランドマーク戦略

- **UI アイコン**: OSS アイコンセット（例: lucide）。**絵文字は使わない**（環境依存で崩れる）
- **イラスト**: 自作 SVG line-art（空状態「まだマップがありません」/ ヒーロー/ ライブ聞き取り中の図）。テーマ色追従（`currentColor` / primary・accent）
- **ブランドマーク（PWA deliverable）**: 単一 SVG ソース → `favicon.svg` / `favicon.ico` / `apple-touch-icon.png`(180²) / `icon-192/512.png` / `icon-maskable-512.png` / `manifest.json` icons + HTML head wiring。モチーフ案: **枝分かれする線（マインドマップの幹と枝）を 1 本のインクラインで**、primary + accent の 2 色。絵文字 favicon は NG
- 写実ラスター: 任意・要キーのエスケープハッチ（MVP では不使用）

## 8. 視覚レビュー基準（Step 4 で適用、UI 実装後）

- 階層・余白・コントラスト・主役（キャンバス）の明確さ・トーンの静けさ
- 人ノード vs AI 提案の視覚区別が成立しているか
- 狭幅（360px）でヘッダー等の水平要素が縦折れしていないか（#2.5）
- デザイントークン適用（生値 hex/px の直書きがないか、#2.6 token-conformance）
- コピー O38 deny-list スキャン
- 入口の「これは何？」理解（O41）
- 課金勧奨の価格透明性（O43、billing 画面）
- ルート到達性（O55、/legal/* 等に導線があるか）
