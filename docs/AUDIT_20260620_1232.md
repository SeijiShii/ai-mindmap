### 3.2 High 詳細

#### [AUDIT-perspective-001] O27/SEC-004 レート制限が未実装
- **検出根拠**: `src/`・`api/` に ratelimit/throttle/429 実装 0 件。論点-008 で `accepted-as-requirement` だが未実装。高頻度逐次マージ (/api/structure /api/expand) のコスト爆発・乱用防御なし
- **推奨アクション**: `/flow:revise ai-structuring` (+ ai-expand) でユーザー/IP レート制限 + 無料枠トークン上限と二重防御 (Upstash Ratelimit 等)。**no-key Class A** (実装可能)

#### [AUDIT-perspective-002] O64 Stripe 課金が実連携されていない (mock のみ)
- **検出根拠**: `stripe` パッケージ未導入 / checkout session 作成 endpoint なし / `api/billing/webhook.ts` ルートなし。`webhook.ts` は idempotent ロジックの unit test のみ。本番で課金フロー (購入→トークン付与) が成立しない + Stripe 署名検証なし
- **推奨アクション**: `/flow:revise billing` で `stripe` 導入 + checkout 作成 endpoint + `api/billing/webhook.ts` (constructEvent で署名検証、不正署名=4xx / 無関係イベント=200 ack / 正当=処理、O64 3 状態)。**ロジック/ルートは no-key Class A**、実キー疎通は release Phase 2

#### [AUDIT-perspective-003] SEC-003/O26 Sentry beforeSend PII マスキングが未実装
- **検出根拠**: @sentry 未導入・Sentry.init / beforeSend 0 件。AI 送信前 scrub (pii-scrub.ts) は実装済だが、エラー監視ログの PII マスキング (論点-007 legal_required) が未配線
- **推奨アクション**: `/flow:revise _shared/ai-client` (or observability target) で Sentry SDK 導入 + `beforeSend` で思考内容/PII マスク + エラーに本文を含めない。**no-key Class A** (DSN は release で FILL、マスキングロジックはコード)

#### [AUDIT-perspective-004] O22(B) 段階認証 (サインイン/連携/サインアウト) UI が未実装
- **検出根拠**: ClerkProvider + 匿名ゲスト ticket は実装済 ((A) PASS) だが、`<SignIn>`/`<UserButton>`/`signOut` ボタン/アカウント連携導線が src/ に 0 件。concept §1.2 が「課金/同期時に段階認証 + ゲスト→アカウント連携データ引き継ぎ」を契約。ゲストが本登録・サインアウト・別端末同期できず、購入トークンがセッション喪失で消失するリスク (CF-20260609-009 broad-match マスク)
- **推奨アクション**: `/flow:revise _shared/auth` (or app-shell) で `<UserButton>` + サインイン/アカウント連携 + サインアウト導線を実装 (両輪)。**no-key Class A**
